# SPEC.md — web-login-validator 詳細仕様書

---

## 1. 概要

「ログインURL・ユーザID・パスワード」の3点セットを入力として受け取り、以下の2段階で検証を行う。

1. **フォーマットバリデーション** — 入力値の形式・完全性チェック
2. **実接続バリデーション** — 実際にHTTPリクエストでログインを試行

---

## 2. 入力仕様

| フィールド | 必須 | バリデーションルール |
|-----------|------|---------------------|
| ログインURL | ✅ | `http://` または `https://` で始まる有効なURL形式 |
| ユーザID | ✅ | 空文字でないこと、全角文字・前後スペースを警告 |
| パスワード | ✅ | 空文字でないこと、全角文字・前後スペースを警告 |

---

## 3. フォーマットバリデーション仕様

### 3-1. URLチェック
- `http://` / `https://` のスキームが存在するか
- ドメイン部分が空でないか
- URLとして構文的に正しいか
- 自動補完候補：`https://` が抜けている場合に提案

### 3-2. ユーザID・パスワードチェック
- 空文字・空白のみでないか
- 全角スペースの混入がないか
- コピペ由来の改行・制御文字の混入がないか
- 前後の余分なスペース（トリム警告）

### 3-3. 種別自動判定
URLのパスやドメインパターンから種別を推定して表示する。

| 判定種別 | 判定条件（例） |
|---------|--------------|
| WordPress管理画面 | URLに `/wp-admin` `/wp-login.php` を含む |
| Xserver サーバーパネル | URLに `secure.sakura.ad.jp` や `panel.xserver.ne.jp` 等を含む |
| その他Webログイン | 上記以外 |

---

## 4. 実接続バリデーション仕様

### 4-1. URL到達確認
- HTTPリクエストを送信してサイトが存在するか確認（HTTP 200 / リダイレクト許容）
- タイムアウト：10秒
- SSL証明書エラーは警告として記録（接続は継続）

### 4-2. ログイン試行
- ログインフォームのフィールドを検出してID/PWを送信
- セッションCookieを保持してログイン後ページを確認
- 判定基準：ログイン後URLが変化している、またはログインページが消えている

### 4-3. エラー種別判定

| エラーコード | 内容 |
|------------|------|
| `URL_NOT_FOUND` | URLに到達できない（DNS解決失敗・タイムアウト等） |
| `LOGIN_PAGE_NOT_FOUND` | URLは到達できるがログインフォームが見つからない |
| `AUTH_FAILED` | ログイン試行したが認証失敗（ID/PW誤り） |
| `MFA_REQUIRED` | ログイン後に二段階認証画面が検出された |
| `LOGIN_SUCCESS` | ログイン成功 |

## 5. 出力仕様

### 5-1. チェック結果サマリー

```
============================
 ログイン情報チェック結果
============================
ログインURL : https://example.com/wp-admin  ✅
ユーザID   : admin                           ✅
パスワード  : ********                        ✅
種別       : WordPress管理画面
----------------------------
接続確認   : ✅ 到達可能
ログイン   : ✅ 成功
============================
```

■ パスワード   :

## 6. 非機能要件

| 項目 | 内容 |
|------|------|
| 実行環境 | CLIツール（ターミナルから実行） |
| セキュリティ | パスワードはターミナル上でマスク表示、ログファイルには保存しない |
| ログ | チェック結果（成否・エラーコード）のみ記録。認証情報は記録しない |
