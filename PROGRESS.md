# Progress (開発進捗)

## 本日の開発ログ (2026-04-01)

### 完了した主要機能
- [x] **2 セクション入力の対応**: WordPress と サーバー管理パネル をセットで入力・検証できるフォーム。
- [x] **WAF 回避 (403対策)**: 「トップページ経由」でのアクセスの再現。
- [x] **WordPress 管理者権限チェック**: ログイン成功後のダッシュボード HTML 解析による権限の自動判別。
- [x] **ログインページの自動探索 (Auto-Discovery)**: トップ URL だけであっても、配下のログイン画面を自動で見つけ出す。
- [x] **デフォルトパスワード表示**: パスワードを伏せず、そのまま情報の共有ができるサマリー機能。
- [x] **一括コピーテンプレート**: 情報を統合してクリップボードに保存する一括レポート機能。

### 解決した課題
- **HTTP 403 エラー (Forbidden)**: ログイン画面に直接アクセスした際、サーバー側のセキュリティに弾かれる現象を、ステップを分けてアクセスすることで解消（Referer と Cookie を使用）。
- **ログインURLの誤認**: SiteGuard 等で URL が変更されている場合でも、URL文字列ではなくコンテンツ（HTML）の内容で判断するように柔軟性を向上。
- **検証パスワードのタイポ**: ログイン送信時のパラメータ名 `wp-submit` の修正。

## 現在のステータス (1.0 完成)

- **実用性**: 「依頼人から情報を受け取り、作業前に確実に裏取りをする」という目的において、ほぼ全ての標準的な WordPress サイトでの検証が可能になりました。
- **課題**: 
  - **二段階認証 (2FA)**: 画像認証やメール認証などが有効なサイトでは、現時点では「認証失敗」または「認証情報の不足」として検出されます。
  - **ログイン先の限定**: 一部の特殊な WAF（IP制限が非常に厳しいもの）については、完全な自動検証ができない場合がありますが、エラー詳細から特定のヒントを提供できるようにしています。

## 次のステップ
1.  **FTP情報の入力**: もしさらに情報を集約する必要があれば、FTP セクションも追加。
2.  **管理履歴の保存**: LocalStorage を使った簡易的な履歴保存機能。
3.  **判定パターンの追加**: 他のレンタルサーバー（さくら、ロリポップ等）の特有のログイン画面の検出。
